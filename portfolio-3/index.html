Work

iOS Mobile Application

Road Fortify

A mobile application for real-time road hazard detection (potholes, debris, animals) powering driver safety and and geotagged hazard report generation for road maintenance

Road Fortify is a native iOS application leveraging a custom YOLOv8 model via CoreML to perform real-time, on-device hazard detection without relying on cloud inference. The architecture combines AVFoundation and CoreLocation to generate precise geospatial telemetry, synchronizing structured data to a centralized PostgreSQL database. This stack utilizes a fault-tolerant queueing system to ensure reliable delivery for downstream analytics and road maintenance prioritization, even in intermittent network conditions.

Web Application

MadMeals

An AI-powered dining hall recommendation system for UW-Madison students. MadMeals analyzes daily menus across campus dining locations and provides personalized meal suggestions based on each student's custom dietary preferences and nutritional goals

MadMeals is a Python-based dining optimizer that aggregates real-time UW-Madison menu data via the Nutrislice API to generate personalized meal plans using LangChain and Google Gemini. It offers both synchronous (Flask) and asynchronous (Quart) implementations to efficiently handle concurrent user loads through non-blocking I/O and thread pooling. The system utilizes Pydantic for strict structured output validation and implements intelligent in-memory caching to optimize performance and minimize upstream API latency.

Distributionally Robust Instruction Fine-Tuning (DRIFT) in LLMs

Advised by Prof. Jelena Diakonikolas • University of Wisconsin-Madison

LLMs suffer from the problem of performance drop when faced with unconventional instruction styles and difficult phrasing. Conventional supervised fine-tuning in LLM post-training instruciton fine-tuning assigns equal weightage to all instruction styles it faces which can lead to poor performance on styles that are hard for LLMs to understand and follow. I am investigating the promise of the Distributionally Robust Optimization (DRO) framework in instruction fine-tuning to dynamically adjust instruction domain weights during the fine-tuning process to make models more robust to difficult instruction styles. Preliminary analysis suggests that not only does this provids more robustness to models, but better overall performance compared to standard SFT as well. A full paper with details is expected in the next 2-3 months.

  • DRO methods in LLM instruction fine-tuning can outperform standard SFT
  • DRO methods can make LLMs more robust to difficult instruction styles
  • Experimentations concieved in PyTorch with use of Huggingface tools and executed on UW-Madison's HTCondor distributed and high-performance computing platform
