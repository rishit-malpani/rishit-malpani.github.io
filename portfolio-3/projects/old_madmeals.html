Overview

MadMeals was born from a common frustration among UW-Madison students: standing in front of a dining hall menu, overwhelmed by choices, and often ending up with the same meals day after day. The platform transforms the dining experience by providing intelligent, personalized recommendations that help students discover new foods they'll love while meeting their nutritional needs.

The system scrapes menus daily from all campus dining locations, analyzes nutritional content, and uses machine learning to match students with meals tailored to their preferences and dietary restrictions.

The Problem

UW-Madison has multiple dining halls serving different menus each day. Students often struggle to find meals that match their dietary restrictions (vegetarian, vegan, halal, gluten-free), track their nutritional intake, or simply discover new options among the hundreds available daily.

Existing solutions were limited to basic menu listings without personalization, nutritional analysis, or intelligent recommendations.

Technical Approach

The backend consists of a Node.js API server connected to a PostgreSQL database. A Python service handles daily menu scraping using BeautifulSoup and performs nutritional analysis using the USDA food database. The recommendation engine uses collaborative filtering combined with content-based filtering to suggest meals.

The React frontend provides a responsive, intuitive interface where students can set preferences, browse recommendations, and track their dining history.

Key Features

  • Smart Recommendations: ML-powered suggestions based on your taste profile and dining history
  • Dietary Filtering: Automatically filters for vegetarian, vegan, halal, gluten-free, and allergen-free options
  • Nutritional Tracking: View calorie counts, macros, and micronutrients for every meal
  • Real-time Menus: Daily updated menus from all campus dining locations
  • Crowd Predictions: Estimated wait times and crowd levels based on historical data
  • Meal Planning: Plan your week's meals in advance and get shopping-list-style summaries

Recommendation Algorithm

The recommendation system uses a hybrid approach combining collaborative filtering (finding users with similar tastes) and content-based filtering (matching meal attributes to preferences). New users start with content-based recommendations based on their stated preferences, gradually incorporating collaborative signals as they rate meals.

I implemented the algorithm using scikit-learn's matrix factorization for collaborative filtering and TF-IDF vectorization for content similarity. The system retrains weekly to incorporate new user data.

Challenges & Solutions

Data Quality: Menu data from the university was inconsistent and sometimes incomplete. I built a robust scraping pipeline with fallbacks and implemented fuzzy matching to handle variations in food names.

Cold Start Problem: New users have no history for the recommendation engine to work with. I addressed this with an onboarding flow that gathers preferences through a quick taste quiz, enabling reasonable recommendations from day one.

Performance: Computing recommendations for thousands of users with hundreds of daily menu items required optimization. I implemented caching strategies and moved heavy computations to background jobs.
